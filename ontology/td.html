Classes

td:ActionAffordance

IRI: https://www.w3.org/2019/td#ActionAffordance

a OWL Class

ActionAffordance - An Interaction Affordance that allows to invoke a function of the Thing, which manipulates state (e.g., toggling a lamp on or off) or triggers a process on the Thing (e.g., dimm a lamp over time).
Sub-class oftd:InteractionAffordance

td:MultiLanguage

IRI: https://www.w3.org/2019/td#MultiLanguage

a OWL Class

MultiLanguage - Container to provide human-readable text in different languages using language tags described in [[BCP47]]. Each used language tags code MUST be decleared as vocabulary term member of this container (e.g., en, de, ja, zh-Hans, zh-Hant).

td:PropertyAffordance

IRI: https://www.w3.org/2019/td#PropertyAffordance

a OWL Class

PropertyAffordance - An Interaction Affordance that exposes state of the Thing. This state can then be retrieved (read) and optionally updated (write). Things can also choose to make Properties observable by pushing the new state after a change.
Sub-class oftd:DataSchema
Sub-class oftd:InteractionAffordance

td:Thing

IRI: https://www.w3.org/2019/td#Thing

a OWL Class

Thing - An abstraction of a physical or a virtual entity whose metadata and interfaces are described by a WoT Thing Description, whereas a virtual entity is the composition of one or more Things.

td:EventAffordance

IRI: https://www.w3.org/2019/td#EventAffordance

a OWL Class

EventAffordance - An Interaction Affordance that describes an event source, which asynchronously pushes event data to clients (e.g., overheating alerts).
Sub-class oftd:InteractionAffordance

td:Link

IRI: https://www.w3.org/2019/td#Link

a OWL Class

Link - A Web link, as specified by IETF RFC 8288 (https://tools.ietf.org/html/rfc8288).

td:InteractionAffordance

IRI: https://www.w3.org/2019/td#InteractionAffordance

a OWL Class

InteractionAffordance - Metadata of a Thing that shows the possible choices to clients, thereby suggesting how clients may interact with the Thing. There are many types of potential affordances, but W3C WoT defines three types of Interaction Affordances: Properties, Actions, and Events.

td:VersionInfo

IRI: https://www.w3.org/2019/td#VersionInfo

a OWL Class

VersionInfo - Carries version information about the TD instance. If required, additional version information such as firmware and hardware version (term definitions outside of the TD namespace) can be extended here.
